Manufacturing Anomaly Detection

Re-evaluation Pipeline Plan — Western Digital Genesis Line

Context

Replacing the existing Auto Vision Setting Tool with a lighter re-evaluation pipeline. Current pipeline is overkill. New one targets the specific case where the existing ML model has already flagged a defect with a red bounding box — we want to re-score it rather than blindly accept the flag.

The Core Idea

  1. Take defect region (bounding box) output from existing model
  2. Crop that region from the raw image (not the labeled result image)
  3. Compare crop against gallery of known-good (pass) images
  4. Produce anomaly score → auto-pass / auto-fail / flag for human review
Keyword: anomaly score via image similarity / anomaly detection

Problems With Naive Similarity Approach

ProblemSeverityFix
Location mismatch — comparing crop to wrong region of reference Critical Gallery must store crops from the same region/position across pass images
Bbox coordinate space mismatch — result image may be resized/padded vs raw Critical Use raw bbox coords from model output, not pixel coords from visualized image
Fixed threshold is fragile — breaks with lighting/camera drift Critical Use dynamic threshold or flag when input is out-of-distribution
Crop size inconsistency — different bbox sizes → different embeddings High Normalize: always crop fixed physical window (e.g. 100×100px) centered on bbox midpoint
ResNet embeddings not domain-aligned — trained on ImageNet, not surfaces High Fine-tune encoder or use PatchCore (builds memory bank from your own data)
No variance model — single threshold ignores natural pass variation Medium Build distribution from 20–50+ pass samples per region, flag by distance from mean

Chosen Approach (Decision Tree)

To be decided based on available pass images and imaging setup.
If imaging is fixed (controlled camera + lighting, product doesn't shift):
Classical Image Differencing
Align crop to reference template, subtract pixels, threshold residual. No ML, fully explainable, fastest.
If fewer than ~50 pass images per region:
SPC on Handcrafted Features
Extract brightness, texture entropy, edge density. Fit multivariate normal distribution. Flag by Mahalanobis distance. No GPU needed, interpretable.
If 50–500 pass images and need pixel heatmap (explainable to QC managers):
Autoencoder
Train on pass images only, flag by reconstruction error map. Shows where the anomaly is.
If 50–500 pass images and need best accuracy (score only):
PatchCore (via anomalib)
Patch-level memory bank from pass images, nearest-neighbor distance at inference. State of the art for pass-only anomaly detection.
Quick prototype / zero data:
CLIP Zero-Shot
Compare crop embedding against text prompts ("defect-free surface" vs "damaged surface"). No training. Less reliable but useful for early testing.
Long-term (add on top of any above):
Human-in-the-Loop Active Learning
Log every human decision on gray-zone items. After ~200–300 reviews, fine-tune small classifier on hard cases only. Gray zone shrinks over time.

Image Generation Prompts (Test Data)

Pass images — defect-free reference gallery

Top-down macro photo of a flat industrial metal surface, uniform brushed aluminum texture, even lighting, no scratches, no discoloration, no marks. Shot from directly above, neutral grey background, photorealistic, 512x512.
Close-up photo of a smooth injection-molded plastic panel, consistent matte finish, uniform color, no bubbles or cracks, factory clean, flat overhead shot, product photography style, sharp focus, 512x512.

Defect images — for testing detection

Top-down macro photo of a flat metal surface with a visible scratch defect — a dark irregular linear gouge running diagonally, surrounded by normal brushed aluminum texture. Defect covers ~20% of image. Red bounding box drawn around the defect region. 512x512.
Overhead photo of a plastic panel with a dark blister/bubble defect — a raised discolored circular anomaly ~3cm wide. Rest of surface is clean and uniform. Red rectangular border highlights the defect. Macro photography, 512x512.

Edge cases — gray zone, for threshold calibration

Overhead photo of a metal surface with a very faint surface mark — could be a minor scratch or a reflection artifact. Slightly different texture in one small region, ambiguous whether it is a defect. No bounding box. 512x512.
Tip: Add "The defect is centered in the image, plain dark background border around edges, consistent fixed camera angle." — makes bbox cropping predictable.

Crop ROI
Normalize to fixed physical window (e.g. 100×100px) centered on bbox midpoint. Ensures consistent embedding size.
DINOv2 Encoder
Self-supervised ViT (Meta). No fine-tuning needed — produces strong patch-level features from facebook/dinov2-base. Processes crop at 224×224 (upsample small crops). Patch size: 14×14px tokens.
Patch Memory Bank
PatchCore: extract features from all pass images, subsample with greedy coreset. At inference, compute nearest-neighbor distance in feature space (via faiss).
Anomaly Score
Max patch distance = image-level score. Per-patch distances = pixel heatmap. Score → AUTO PASS / AUTO FAIL / GRAY ZONE.

Entire image fed to ViT — not just the crop. Captures product position, surrounding area, lighting context.
ViT CLS Token
DINOv2's CLS token = global image summary embedding. Encodes "what region of the part, what surrounding context." No spatial info lost.
Global Embedding
768-dim (DINOv2-base) or 1024-dim (large). Store one embedding per pass image. Build distribution (mean ± covariance). Flag if test embedding is out-of-distribution.
Crop ROI
Same as Architecture A — normalized fixed-window crop centered on bbox.
Local Features
DINOv2 patch features on crop, or PatchCore memory bank. Produces local anomaly score.
Local Score
0.0 = normal crop, 1.0 = anomalous crop. Same as Architecture A output.
Fusion Layer
Options: (1) weighted sum — final = α·local + (1-α)·global_ood. (2) learned gating — train a tiny MLP on global embedding to modulate local score. (3) rule-based: if global OOD score high → escalate to GRAY regardless of local.
Final Score
Combined signal. Example: local says PASS but global says image looks unusual → escalate to GRAY. Reduces false passes from lighting drift or positioning shift.

Full Raw Image
No preprocessing crop. Full resolution image resized to 518×518 (DINOv2 optimal) or 224×224.
DINOv2 Self-Attention
Each layer computes attention between all patch tokens. DINO's self-supervised training makes attention heads naturally focus on semantically meaningful regions — known to highlight object/defect boundaries without labels.
Attention Rollout
Aggregate attention across all heads and layers → per-patch relevance map. Upscale to original resolution = pixel-level heatmap. Hot regions = ViT focused here. Useful for: QC managers, auditing, active learning labels.
Anomaly Regions
Threshold attention heatmap → candidate anomaly masks. Compare patch features at hot regions vs memory bank of pass images. High distance = anomalous patch.
Score + BBox
Image-level anomaly score + auto-generated bounding box from attention mask. Can feed back into existing workflow as bbox source — replaces or validates existing model's bbox.
Note: Arch C is the most ambitious — it replaces the existing model's detection role, not just the re-scoring role. Validate Arch A or B first.
No dependency on existing model's bbox quality

Replaces more of the existing pipeline — higher risk
